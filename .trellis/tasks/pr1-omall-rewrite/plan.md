## 4. 关键决策

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 路由 | hash 路由 | CloudBase 静态托管刷新子路径不 404，无需服务端 rewrite |
| 数据来源 | 先本地静态数据 | 不被未完成的 API/DB 阻塞，先打通可展示闭环 |
| 状态管理 | 优先轻量（Context / 小型 store） | 购物车是唯一跨页状态，避免过度引入依赖 |
| 图片 | 复制进 `web/public/images/` | 当前项目独立构建发布，运行时不引用 `shopTwo-main` |
| 后端/DB/支付 | MVP 后置 | 阶段一只交付 CloudBase 静态商城 |

## 5. 风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| 仍用 path 路由就发布 | 详情页刷新 404 | M1 先切 hash 路由，build+preview 验证 |
| 照搬 `shopTwo-main` 代码/包名 | 污染重写项目 | 只借鉴结构与交互，包名用 `@o-mall/*` |
| 文档把未完成写成已完成 | 误导后续开发 | 严格对照代码；未发布只写"可发布" |
| 静态资源未进 `dist` | 线上图裂 | 确认图片在 `public/images/`，build 后查 `dist/images/` |

## 6. 不做（本 MVP 之外）

真实支付、完整账号体系、管理后台、真实 PostgreSQL 商品/订单持久化、复制/修改 `shopTwo-main`、把腾讯云 Linux 服务器作为第一发布目标。
