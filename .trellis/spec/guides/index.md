# Trellis 指南索引

实现功能或做较大文档改动前，先读这个文件。

## 开发前检查

- 确认改动是否影响：
  - 后端路由行为
  - 前端用户流程
  - 数据库 schema
  - OpenAPI 合同
  - CloudBase 静态托管部署
  - workspace/package 配置
- 继续读取 `../workspace/index.md`。
- 跨层改动必须同时更新实现和合同文档。
- 当前阶段 CloudBase 静态发布优先，不要把腾讯云 Linux 服务器当作第一发布目标。
- `shopTwo-main` 只作参考，不是当前项目的真源。

## 思考规则

- 先看功能树，再看文件职责。
- 数据流要写清楚：
  - 浏览器 UI
  - API 路由
  - 校验
  - 数据库操作
  - 响应格式
- 商城功能优先合同先行。
- 文档中的路径和命令必须对照当前文件验证。
